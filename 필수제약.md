# LocalHub Project Rules

## Project Overview

- Project: LocalHub (지역 정보 공유 커뮤니티)
- Frontend: Vue.js 3 (Vite) SPA
- Backend: 없음
- Data Source: 제공된 JSON 파일만 사용
- Deployment: Netlify

---

# Mandatory Requirements

## Data

- 제공된 JSON 파일을 프론트엔드에서 직접 사용한다.
- 공공 API를 직접 호출하지 않는다.

## Community

- 익명 게시판만 구현한다.
- 회원가입 및 로그인은 구현하지 않는다.
- 게시글 CRUD를 구현한다.
- 게시글은 localStorage에 저장한다.
- 수정/삭제는 비밀번호 검증으로만 처리한다.

## Chatbot

- OpenAI API를 사용한다.
- Vue3 프론트엔드에서 OpenAI API를 직접 호출하는 방식으로 구현하며, API 키는 환경변수(.env, VITE_ 접두사)로 관리함 — 제공 JSON 데이터 기반 자연어 지역 정보 질의응답 처리
※ 주의: Vite의 .env(VITE_ 접두사) 값은 빌드 결과물에 그대로 포함되어 브라우저에서 API 키가 노출될 수 있음. 반드시 사용량 제한이 걸린 키만 사용하고, 결제 한도를 낮게 설정할 것
- 관광지, 축제, 음식점, 게시글 검색 등의 질문에 답변한다.
- 채팅 히스토리를 유지한다.
- 모바일 대응 UI를 제공한다.
- 플로팅(Chat Floating Button) 형태로 구현한다.

## Frontend

- Vue.js 3 + Vite SPA 구조를 사용한다.
- 새로고침 없이 페이지를 이동한다.

## Deployment

- Netlify에 배포한다.

---

# Constraints (Must Follow)

## Data

- 추가 데이터 사용 시 반드시 라이선스를 확인한다.
- 공공누리 유형을 확인한 후 사용한다.

## OpenAI

- API Key는 `.env`의 `VITE_` 변수만 사용한다.
- 사용량 제한(Key Limit)이 적용된 Key만 사용한다.
- API 비용은 예산을 초과하면 안 된다.

## Git

- `.env` 파일은 Git에 업로드하지 않는다.

## Development

- VSCode + GitHub Copilot + OpenAI API만 사용한다.
- Claude Code, Cursor, Codex 등은 사용하지 않는다.

## Scope

- 개발 중 기능을 추가하지 않는다.
- MVP 범위를 변경하지 않는다.

---

# Development Rules

- Component 기반으로 개발한다.
- 공통 컴포넌트를 우선 사용한다.
- Router 기반 SPA를 유지한다.
- 중복 코드를 만들지 않는다.
- 유지보수가 쉬운 구조를 작성한다.

# Do Not

- 회원가입 구현
- 로그인 구현

- Database 구축
- 공공 API 직접 호출
- MVP 외 기능 추가
